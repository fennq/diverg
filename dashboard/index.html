Blockchain Investigation (Primary)

Main workflow for wallet, contract, and token-bundle investigations

On-chain

Three separate tools: fast RPC check, deep analyst report, or Solana token-holder bundle.

1 · Quick check
Wallet or contract snapshot

Balance, Solana account shape, recent signatures (or EVM nonce + optional Etherscan txs). Fast — uses your Helius key from Settings for Solana.

Ethereum 0x… uses public RPC; if the server has ETHERSCAN_API_KEY, recent mainnet transfers appear in the summary.

Enter an address and run quick lookup.
Full JSON

              

            


            

              
2 · Analyst report

              
Full chain investigation

              
Uses the address in step 1 as the main wallet/contract/deployer (or paste extra token mints/contracts below). Runs the server blockchain_investigation skill — not a court-ready report; intelligence for authorized follow-up.

              

                What you’ll see in the results panel
                
    
                  
  • Verdict line — risk band (high / elevated / moderate-low) plus short reasons (e.g. CEX exposure, high-severity findings).
    • 
                  
  • Summary — numeric risk score (0–100), finding counts, evidence quality; may note sniper/LP/CEX themes when data supports them.
    • 
                  
  • On-chain data — one line from the server on whether Solscan/Etherscan data was used or skipped (keys missing).
    • 
                  
  • Focus address / deployer — address, label, explorer button, named counterparties when labels exist.
    • 
                  
  • Tokens tied to this pass — discovered mints/contracts and mint/freeze-authority warnings when present.
    • 
                  
  • Chronology — dated money-flow lines from transfer edges when the graph is populated.
    • 
                  
  • Flow highlights — counterparty notes, CEX routing, outflow snippets.
    • 
                  
  • Red flags — checklist-style signals (LP removal, mint authority, etc.) — heuristics, not accusations.
    • 
                  
  • Explorer links — Solscan / Etherscan (or chain explorer) for graph-related addresses.
    • 
                  
  • Flow edges + Mermaid — sample hops; use "Copy flow as Mermaid" for diagrams elsewhere.
    • 
                  
  • Findings — full structured list (severity, evidence, source) under the summary blocks.
    • 
                

              

              
If SOLSCAN_PRO_API_KEY / ETHERSCAN_API_KEY (and related keys) are missing on the server, on-chain sections will be thin or skipped — check Full JSON → crime_report.data_sources and errors.
