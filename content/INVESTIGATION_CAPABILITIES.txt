DIVERG — WHAT WE CAN AND CAN'T GATHER (HONEST ASSESSMENT)
===========================================================
Use this to set expectations for "investigate this wallet with prompt" and similar requests.

BLOCKCHAIN — STRONG (on par when API keys set)
-----------------------------------------------
• Transfers, counterparties, flow graph (chronological), deployer/token context.
• Solscan (Solana) + Etherscan (Ethereum) + Arkham (labels, CEX/mixer tagging).
• Bubblemaps for token holder clusters and relationships.
• Crime report: verdict, risk score, red flags, chronological narrative, explorer links.
• No Twitter or general web data is used here — chain only.

WEB (general search) — ADDED VIA run_web_search
-------------------------------------------------
• run_web_search(query) runs a DuckDuckGo search and returns title/snippet/URL for each result.
• Use it to search for: wallet address + user prompt, token name + scam, project name + rug, etc.
• The bot can call run_blockchain_investigation (wallet) + run_web_search(wallet + " " + prompt) and collate.
• This is public web search only (no authenticated Twitter, no paywalled sources).

TWITTER / X — COVERED VIA run_x_search
---------------------------------------
• run_x_search(query) searches X for tweets matching the query (e.g. wallet address to link wallets to X accounts).
• When X_API_BEARER_TOKEN is set: uses X API v2 recent search (reliable, rate-limited). Get token from developer.x.com.
• When NITTER_BASE_URL is set (e.g. https://nitter.poast.org): fallback scrape of Nitter search page. Many public instances are rate-limited or down.
• Returns: tweet text, author_username, url (x.com/username/status/id), created_at — use to link wallets to X accounts and collate with chain + web.
• We do not: read DMs, full timelines, or private content — only public search results.

ENTITY REPUTATION (company/person names)
----------------------------------------
• run_entity_reputation(domain) uses WHOIS org/registrant + DuckDuckGo to find fraud, lawsuits, breaches, FTC/SEC.
• Only applies when we have a domain (so we get entity names from WHOIS). For "investigate wallet 0x123" we have no domain — unless we get a label from Arkham (e.g. "Binance") and then search for that entity.

BOTTOM LINE
-----------
• "Investigate wallet X with prompt": we can do blockchain + run_web_search + run_x_search and collate. Set X_API_BEARER_TOKEN (or NITTER_BASE_URL) to enable X search and link wallets to X accounts.
• X search returns public tweets only (no DMs or private content). Use author_username and tweet url in the report to show wallet ↔ X links.
