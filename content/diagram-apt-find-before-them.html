Find Before They Do
APT-style upgrades: code intel, CVE awareness, logic hunting, chain-driven next tests
1. Client-side code intel
Read the frontend
Source maps, JS fetch/axios → API shape. Dangerous sinks (eval, innerHTML). Sensitive storage keys. Version hints for CVE.
2. Dependency & CVE
Know the stack
Versions from headers + JS. Curated watchlist (Next.js, React, Express…). “Detected X.Y; CVE-XXXX may apply.”
3. Logic & numeric hunting
Think like an attacker
Amount/limit/offset: 0, -1, MAX_INT, NaN. Bounds bypass. Refund/rounding. CONFIRMED when server accepts.
4. Chain-driven next tests
If A then try B
Verbose error → recommend SSRF. IDOR + JWT → recommend user_id sub. GraphQL → batch/depth. In report: RECOMMENDED NEXT TESTS.
Outcome: Same classes of issues sophisticated actors find — code, supply chain, logic, chaining — surfaced earlier.
