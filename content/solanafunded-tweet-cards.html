Security Assessment

CASE FILE: SOLANAFUNDED.COM

Subject
solanafunded.com — Funding / crypto platform. Cloudflare front · TLS 1.2 · Certificate valid (Google Trust Services). Full scan: headers, crypto, exposure, web vulns, payment surface.
Infrastructure
Server: Cloudflare. HSTS present but misconfigured (missing includeSubDomains). Multiple storage paths exposed: /uploads, /storage, /assets, /files, /media, /downloads. Debug toolbar path present.
Scope
Full scan · 17 skills. 30 findings: 3 High, 6 Medium, 14 Low, 7 Info. Company surfaces: 7 (debug, storage dirs). Platform handling funds — expectations for security are high.

Diverg assessment · March 2026

DIVERG
Evidence — High

Dangerous paths & HTTP methods

1 phpMyAdmin — https://solanafunded.com/phpMyAdmin/ reachable. PUT, PATCH, DELETE enabled without auth. Database admin surface exposed; unauthenticated modification/deletion possible.
2 /admin/ — Admin route publicly reachable. PUT, PATCH, DELETE enabled without auth. Management surface exposed; takeover and data loss risk.
3 /__debug__/ — Framework debug toolbar path reachable. PUT, PATCH, DELETE enabled without auth. Stack and internal state disclosure; debug endpoints must not be in production.
Impact
Unauthenticated users could modify or delete resources on all three paths. For a platform that handles funds, exposing phpMyAdmin and admin with dangerous methods is unacceptable.

Diverg assessment · March 2026

DIVERG
Evidence — Medium

Missing & misconfigured security headers

1 Content-Security-Policy — Missing. No XSS mitigation at browser level; injection and data injection risk.
2 X-Frame-Options — Missing. Clickjacking possible; site can be embedded in malicious frames.
3 Strict-Transport-Security — Misconfigured. Has max-age but missing includeSubDomains; subdomains not forced to HTTPS.
Also missing
X-Content-Type-Options, Referrer-Policy, Permissions-Policy, Cross-Origin-Opener-Policy, Cross-Origin-Resource-Policy. Server header reveals Cloudflare.
Frontend crypto
Math.random() used in frontend (predictable). If used for tokens/sessions, impact is high. Use crypto.getRandomValues().

Findings summary

30 findings — 3 High, 6 Medium, 14 Low, 7 Info

High (3)
Dangerous HTTP methods without auth on phpMyAdmin, /admin/, /__debug__/. Broken access control (OWASP A01).
Medium (6)
No CSP, no X-Frame-Options, HSTS misconfigured, weak frontend crypto (Math.random). Security misconfiguration (OWASP A02 / A04).
Low + Info (21)
Missing Referrer-Policy, Permissions-Policy, cookie flags, Server disclosure; TLS/cert checks pass. Defense-in-depth gaps.
Conclusion
For a platform that handles funds, solanafunded.com ships with database and admin surfaces exposed, dangerous methods enabled, and key browser security headers missing. Lock down phpMyAdmin and admin; remove or protect __debug__; add CSP and X-Frame-Options; fix HSTS.
