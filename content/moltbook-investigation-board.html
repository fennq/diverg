Evidence

Weak crypto — session & token risk

Finding
Math.random() used in frontend JS — predictable, not cryptographically secure. If used for tokens, session IDs, CSRF nonces, or security-sensitive values → predictable output.
Affected assets (full URLs)
https://moltbook.com/_next/static/chunks/785528a67640bd6e.js
https://moltbook.com/_next/static/chunks/b236f1f448afb5e0.js
User impact
Predictable or weak crypto can lead to session prediction, token forgery, or password cracking. Session hijack, account takeover, or abuse of recovery/token flows.
Remediation
Use crypto.getRandomValues() for any security-sensitive randomness. Remove Math.random() from auth/session/token flows. Avoid MD5/DES/RC4/ECB for security-sensitive data.
