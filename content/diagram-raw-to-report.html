From Raw to Report
How tool output becomes a prioritized, evidence-backed report
INPUT
Raw results
JSON per skill: findings, header_findings, ssl_findings, extracted_endpoints, detected_versions, errors. Keyed by e.g. headers_ssl:full, client_surface:full.
Correlation
Collect & classify
All findings → buckets (Data Exposure, Access Control, Auth, Operational, Exploitation, Transport…).
Rank & attack paths
Exploitability score. Likely attack paths (e.g. “Info disclosure → SSRF”). Evidence finding IDs.
Recommended next tests
If verbose error → SSRF. If IDOR + JWT → user_id sub. If GraphQL → batch/depth. If client_surface endpoints → API discovery.
Quality & readiness
Coverage, confidence, not_fully_tested. Readiness score for “how production-ready is this target?”
LLM analysis
Context: Correlated context (buckets, ranked_findings, likely_attack_paths, recommended_next_tests) + truncated raw results.
Report: Top findings, attack paths, RECOMMENDED NEXT TESTS, detection/readiness. CONFIRMED vs POSSIBLE. Source tags per claim. No sensitive data in evidence.
