Confidential — Security Assessment

CASE FILE: LIQUID.AF

Subject
liquid.af — Solana launchpad (token creation / listings). Subdomains: www, devnet, metadata, docs. Cloudflare front · Let's Encrypt. Domain and subdomains in scope for recon and chain follow-up.
Infrastructure & DNS
Nameservers: Cloudflare. Stack: Next.js, React, Node. Broad subdomain surface. Certificate valid; HSTS/CSP present but TLS 1.0/1.1 still accepted (see Exposure).
Assessment type
Full scan · Website + blockchain. Preset: full. Target profiles: admin-surface, debug-surface, identity-surface, crypto-launchpad. Block chain: Solana (deployer, token flows, counterparties).

Diverg assessment · 12 March 2026

Evidence

Website exposure — public reach

1 Admin — /admin, /console, /dashboard, /backoffice → 200. No auth enforced. Management routes publicly reachable; credential and takeover risk.
2 Identity — jwks, openid, /sso, /login, /oauth public. Token and session endpoints exposed; enumeration and auth-bypass testing possible.
3 Debug / observability — actuator, __debug__, metrics, health, grafana, kibana reachable. Stack and availability disclosed to attackers.
4 API docs — swagger, openapi, graphql/schema public. Endpoints and parameters discoverable; increases attack surface for IDOR and injection.
5 TLS — 1.0 / 1.1 still accepted. Weak crypto; downgrade and MITM risk. Recommend TLS 1.2+ only.

Evidence

Blockchain — launchpad risk

Platform
Launchpad (Solana). Sniper activity, LP removal risk, fee abuse. Deployer counterparties and token flows mapped when on-chain data available. High risk for rug and wash-trading patterns.
Findings
Token/transfer, account/transfer, holders, defi activity. Connected wallets identified and mapped to primary and counterparties (CEX, mixer). Flow graph: nodes = addresses (primary, wallet, counterparty); edges = amount, unit, date (full YYYY-MM-DD).

Case summary

liquid.af — findings

Web
Admin, identity, debug, API docs public. TLS 1.0/1.1. Broad attack surface; management and internal endpoints reachable.
Chain
Launchpad risk. Connected wallets mapped; deployer → wallets → CEX/mixer. One funder → many wallets = red flag.
Conclusion
Negative findings on both web and chain. Lock down admin and debug; restrict TLS to 1.2+; review deployer and linked-wallet flows. All timestamps in flow use full date (day, month, year).
