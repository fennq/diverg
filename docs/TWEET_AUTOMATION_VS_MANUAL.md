## Single post (copy-paste)

Manual thread-grade work (explorers, tabs, serious clock) optimizes for **story + proof**.

Diverg optimizes for **first structured signal**:

• sketch token + cluster + bridge hints: often **minutes** vs **hours** to build the same shape by hand  
• re-check after deploy / mint: **re-run + diff** vs redoing the trace  
• **more targets in parallel** vs one narrative at a time  

Different job. Automation = triage speed + throughput; humans = final call + thread.

@DivergSec

## Thread (4 posts)

**1/** Everyone knows the manual masterclass: one investigator, infinite tabs, a thread that took real time. That’s not what we’re trying to replace.

**2/** Diverg is the **first mile**: holders, funders, coordination heuristics, bridge flags — machine output in **~minutes to low tens of minutes** (keys, scope) vs **hours** to sketch the same map by hand before you know if the case is worth it.

**3/** That’s **throughput**: more leads screened, faster no, faster escalate to human deep dive. Plus **scan diff** — what changed — without replaying the whole manual pass.

**4/** We’re not claiming automation beats a **finished** public investigation on narrative. We’re claiming it beats **starting from zero** on speed to a structured picture. Authorized use; verify on-chain.
